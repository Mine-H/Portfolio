Project View:

Portfolio

Overview

Coded in HTML, CSS, JavaScript, and LESS, this project has been ongoing since late 2024. The experience of creating and maintaining this webpage has taught me a lot of these languages that I would otherwise not have come across, and it still gives me great insights into User Experience considerations on every iteration of the page.

Understanding that recruiters, colleagues, and curious aquaintances do not have the time to dive through all I've contributed to, I designed and programmed this website. Its purpose is to serve as a platform for showcasing my skills as a Games Designer and Programmer in a more digestible fashion.

Initially organised with collapsible categories of projects, where each category was organised by "latest", the current Portfolio maintenance aims to rework that into a more grid-like, "Home Feed"-inspired approach for a better user experience.

Development

Initially designed for HTML/CSS/JS only, the page's style started to slow down the project's overall development speed. This pushed me to move the style's source code to a language that handles Object Oriented Programming -like syntax, with added benefits including in-line maths and variables with (subjectivelly) better syntax than raw CSS. This was around the time I created the homepage's title card, the code of which was noticeably more organised than the rest of the page's stylesheets and has become the standard I now hold any source code additions to.

A major design shift happened around mid-2025, where I noticed mobile usage of the Projects Showcase section involved a lot of scrolling for not much detail communicated. The observation led me to begin drafting the layouts for individual project pages (thus removing clutter information from the showcase) and improving the project navigation, which resulted in the modern Project Card design present in the homepage. I also set myself a deadline to complete these updates by: the 2026 GradEx, around the end of May and beginning of June.

Most other small improvements came in the form of User Experience considerations (such as the floating tooltip to "Click or Scroll" in the homepage's arrow, in case the user had not noticed the arrow appearing) or file compression to shorten the time the page's critical information takes to load.

Iterations

Picture 1: the previous Projects Showcase page.

The headings were collapsible to allow searching in just one engine's category, and the descriptions were kept short to prevent cluttering. On mobile, the images would appear on top of their headers instead of beside them. This, together with the smaller screen width, resulted in the experience involving a lot of scrolling.

The shorter descriptions also meant I would not be able to go this much in detail into any one decision made in any project. Instead, each project would get an overall view and no deeper dives. This is not ideal, especially in a professional context where those interested in my work were not able to find out why a feature was implemented a certain way and very little of how it was implemented in the first place.

Picture 2: the Homepage title screen without the tooltip.

When user-testing the then-recently-implemented scroll-down arrow on a mobile device, the tester waited for the arrow to appear every time the page loaded, then click it to continue exploring the page rather than scrolling. This was an unintended consequence of the arrow being clickable, and a similar incident has not occurred since adding the clarification "Click or scroll to continue".
